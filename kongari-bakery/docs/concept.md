# こんがりベーカリー(仮題)— 企画書

## コンセプト (v0.7〜)
**ずんだもんと14日間のパン屋経営に挑むラン制ローグライト。**
毎朝カードを1枚ドラフトして店内グリッドに配置し、隣接シナジーでビルドを組み、
毎晩上がり続ける家賃と戦う。生き残り方によってエンディングが分岐する。

### コアループ
1. 朝: 3枚から1枚ドラフト → 4×3グリッドに配置(隣接シナジーが生産を決める)
2. 昼(120秒): 自動生産+手焼きクリック。注文・ゴールデンパン・天気イベント
3. 夜: 家賃支払い(毎日×1.42で上昇)。払えなければ倒産=夜逃げエンド
4. 14日目の決算で、ビルドの偏りに応じた7種のエンディング

### 戦略の源泉
- カード14種×タグ7系統(機械/職人/甘味/緑/接客/黄金/手仕事)の隣接・全体シナジー
- 「生産を伸ばす」か「注文・名声で稼ぐ」か「クリック特化か」のビルド分岐
- 家賃曲線がDPSチェックとして機能し、引きと配置のリカバリーを迫る

## ターゲット体験
- 1プレイ数分でも気持ちいい(クリック→数字が伸びる→設備購入のループ)
- 放置しても進む(オフライン進行は将来課題。まずはタブを開いている間の自動生産)
- スクショを撮りたくなる賑やかな見た目(SNS拡散狙い)

## コアループ
1. パンをクリック → パンが増える
2. パンを使って設備(オーブン、パン職人、支店…)を購入
3. 設備が自動でパンを生産 → より高い設備へ
4. 実績・見た目の変化で達成感

## スコープ(v1 = unityroom公開版) ※実装済み
- クリック生産 + 自動生産設備4種 + クリック強化2種(投資配分の悩み)
- 購入するたびコスト上昇(×1.15 / クリック系は×1.6〜1.8)
- **注文システム**: 時間制限付きの納品依頼。在庫を納品に使うか設備に使うかの判断。報酬は割増パン+名声☆(名声は生産倍率+2%/☆)
- **サプライズ**: ゴールデンパン(横切る/ボーナス)、クリティカル(6%/×10)、こげパン(4%)
- **クリア要素**: 累計10万個で物語の結末(エンディング)
- **マルチエンディング6種**: プレイスタイルで分岐(焦がし/黄金/手焼き/接客/全自動/ふつう)。未達エンドは「??? + ヒント」で一覧表示
- **トロフィー12種** + 称号トースト
- **周回(新しい朝)**: エンディング後にリセット可。見たエンド数に応じてクリック+1・生産+5%の引き継ぎボーナス
- **営業日システム(v0.4.0〜)**: 1日=120秒のリアルタイム営業。日替わりの天気(晴れ=ゴールデン増/雨=注文減・報酬増)。閉店時に経営レポート(手焼き/自動の内訳、注文成績、名声変動)を表示し、翌日の経営方針を選択(品質重視/量産体制/接客強化/ふつう)。放置中は同方針で自動継続
- 効果音・BGM(自作プログラム合成)、ミュートボタン
- エラーログ機構(ゲーム内ログ+JS側捕捉+異常終了検知、タイトル5回クリックでビューア)
- セーブ(ブラウザ内自動保存、周回をまたいでトロフィー/エンド保持)

**v2以降(BOOTH・Steam版の差別化要素):**
- オフライン進行、図鑑、イベント・期間限定要素、追加エンディング

## アート方針
- フリー素材を活用(Kenney、いらすとや等、ライセンス確認の上)
- 足りない分は絵文字・図形で代用 → 後から差し替え
- 解像度 1280×720、ブラウザ埋め込み前提

## 技術
- Godot 4.6 / GDScript / Compatibilityレンダラー
- Webエクスポートは**スレッド無効**(unityroom要件: GDExtension・Thread Support非対応)
- セーブは user:// (WebではIndexedDBに保存される)

## リリース計画
1. unityroom(無料・Web)で公開、反応収集
2. itch.io ミラー(任意)
3. BOOTH: DL版無料 + 支援版(サントラ/設定資料/追加コンテンツ)
4. 反応が良ければ Steam 完全版(プレステージ、実績、オフライン進行を追加)
